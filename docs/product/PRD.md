# ClearMate - 产品需求文档 (PRD)

## 一、项目定位

**ClearMate** — 一个帮助普通人看懂风险、处理麻烦、维护权益、节省金钱和时间的 AI 生活事务代理平台。

它不是普通 AI 聊天助手，而是面向大众生活场景的任务型 AI 系统。用户不需要学习 prompt，只需要上传截图、文件、账单、合同、聊天记录，或描述自己遇到的麻烦，系统就能自动分析、归类、给出风险判断、生成行动方案，并把事情沉淀成一个可跟进的任务。

## 二、核心场景

1. **防诈骗判断**：判断短信、聊天记录、广告、兼职、投资项目是不是诈骗或套路。
2. **文件解读**：解读合同、账单、通知、PDF 文件，提取关键金额、日期、风险条款。
3. **维权材料生成**：帮用户生成退款、投诉、取消订阅、客服沟通、维权材料。
4. **事务管理**：帮用户管理账单到期、合同到期、退款跟进、证件提醒。
5. **长期扩展**：普通人的 AI 消费权益代理、文件管家、风险助手和生活任务中心。

## 三、产品原则

1. 不做"聊天框套壳"，而是做"任务系统"。
2. 普通用户不需要理解 AI，系统负责把用户问题转成结构化任务。
3. 输出必须通俗、明确、可行动，不能像论文。
4. 涉及法律、医疗、金融、投资、合同等高风险内容时，必须标注风险和免责声明。
5. 高风险操作必须要求用户确认（发送邮件、联系商家、提交投诉、删除文件等）。
6. 用户隐私优先，敏感信息要尽量脱敏展示。
7. 每一个分析结果都要区分：事实、推测、建议、待核实事项。
8. 工程上必须可维护、可测试、可部署、可扩展。

## 四、目标用户

| 用户群 | 核心需求 |
|--------|---------|
| 学生 | 看通知、处理课程文件、避免兼职诈骗、写投诉或申请材料 |
| 上班族 | 处理账单、订阅、退款、合同、邮件、消费纠纷 |
| 家庭用户 | 管理保险、房租、学校通知、老人防骗、家庭账单 |
| 老人/数字弱势 | 简单、清晰、大字体、安全提示 |
| 小商户/自由职业者 | 处理客户沟通、报价、投诉、售后、合同草案 |

## 五、核心功能模块

### 1. 用户系统
注册登录、用户资料、家庭空间、成员管理、老人模式、儿童模式、权限控制、隐私设置、操作授权记录。

### 2. 文件与证据系统
上传图片/PDF/Word/Excel/文本、OCR 识别、PDF 文本提取、文件分类、文件摘要、关键日期/金额/相关人提取、敏感信息识别与脱敏、证据链整理、时间线生成、文件历史版本。

### 3. AI 分析系统
防诈骗分析、合同风险分析、账单异常分析、订阅识别、购物避坑分析、退款理由生成、投诉材料生成、客服沟通策略、文件通俗解释、行动清单生成。

AI 输出结构化格式：
```json
{
  "summary": "一句话总结",
  "risk_level": "low | medium | high | critical",
  "risk_points": [],
  "key_facts": [],
  "assumptions": [],
  "suggested_actions": [],
  "questions_to_verify": [],
  "disclaimer": ""
}
```

### 4. 任务流系统
每一件用户事务变成一个 Task，包含完整状态机（draft → pending_info → analyzing → waiting_confirmation → ready_to_execute → in_progress → waiting_response → completed/failed/archived），每个任务有标题、类型、状态、风险等级、关联文件/分析/行动步骤、操作日志、时间信息。

### 5. 行动生成系统
MVP 生成可复制行动材料：投诉信、退款申请、客服话术、邮件草稿、电话沟通提纲、证据清单、下一步行动计划。后续扩展自动发送邮件、浏览器自动化等。

### 6. 风险系统
诈骗风险、隐私风险、财务风险、合同风险、消费陷阱、投资风险、未成年人风险、老人防骗风险、法律不确定性、信息不足风险。每次分析给出风险等级、原因、可疑信号、建议/不建议操作、需核实信息。

### 7. 提醒系统
任务截止提醒、退款跟进提醒、合同到期提醒、账单付款提醒、订阅续费提醒、证件过期提醒、文件待处理提醒。MVP 先做站内提醒，后续做邮件/短信/推送。

### 8. 管理后台
用户管理、文件管理、任务管理、风险规则管理、Prompt 模板管理、模型调用日志、系统成本、用户反馈、错误日志。

## 六、MVP 三个核心入口

1. **这是不是坑？** — 上传截图或输入文字 → 创建任务 → 提取文本 → 分析风险 → 输出风险等级 + 可疑点 + 建议
2. **帮我退款/投诉/取消** — 输入问题 → 创建任务 → 补充信息 → 生成证据清单/退款申请/客服话术/跟进计划
3. **帮我看懂这份文件** — 上传文件 → 创建记录 → 提取文本 → 判断类型 → 生成通俗摘要 + 关键信息 + 风险条款 + 行动清单

## 七、UI 风格

- 简洁、普通人友好、不要技术感太重
- 风险等级醒目，输出像"人话"
- 老人模式：大字号、高对比、少按钮
- 按钮文案直白：开始检查、上传截图、生成投诉信、复制文案、设置提醒、标记已处理

## 八、安全与合规

1. 所有 AI 分析必须加免责声明
2. 不能替代律师、医生、金融顾问
3. 不能诱导用户进行违法行为
4. 不能直接承诺一定退款成功
5. 涉及转账、投资、验证码、身份证、银行卡时必须强提醒
6. 敏感信息展示时部分遮盖
7. 文件删除要二次确认
8. 自动执行动作必须要求用户最终确认
9. 模型调用记录日志但不记录明文敏感信息
10. 开发环境可用 mock，生产不能依赖 mock

## 九、长期扩展路线

1. 消费者权益代理（订阅识别、自动取消、账单异常检测、退款统计）
2. 购物避坑助手（商品分析、评价真假判断、退货窗口提醒）
3. 家庭文件 OS（共享文件库、证件到期提醒、房租合同管理）
4. 老人防骗模式（大字体 UI、一键发给家人确认、高风险弹窗）
5. 自动执行代理（邮件发送、表单填写、浏览器自动化、自动跟进）
